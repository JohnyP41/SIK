Przyklad 3
\\\
Wysylamy 10000 bajtów.Nawiazujemy polaczenie TCP.
Dane sa wysylane w segmentach o dlugosci 1,5kb.
Najpierw mamy flagę SYN odbiorca i nadawca sie synchronizuja.
Potem sa wysylane zostaja segmenty z flaga ACK.Segment doszedł do odbiorcy.Flaga ACK to potwierdzenie dostarczenia.

Przyklad 4
\\\
Wysyłamy 1000 bajtów protokołem TCP.
Nie ma fragmentacji.
Dane wysłane są w jednym 1000 b, segmencie. Na końcu zostaje wysłane potwierdzenie (jeśli segment dotarł do odbiorcy) z flagą ACK.

Przyklad 5
\\\
Wysylamy 1000 bajtów protokołem UDP.
Następuje fragmentacja na poziome IP.
Nie mieszczacy sie pakiet jest ciety na kawałki.
Offsety maleją. Flagi sa postaci 0x03 i są takie same.
Kawałki sa wysylane od konca.

Przyklad 6
\\\
Nie ma fragmentacji.
Wysylamy 1000 bajtów protokołem UDP w jednym datagramie.

Przyklad 7
\\\
Numery sekwencji to kolejno 0,0,1,1,1,2
Numery akceptacji to kolejno 1,1,1,2,2
Wystepuja krótkie komunikaty z flaga ACK i FIN.

12:10:09 <p>
12:10:09 The TclHttpd is a pure Tcl implementation of a Web server.
12:10:09 It runs as a Tcl script on top of
12:10:09 <a href=http://www.tcl.tk/software/tcltk/>Tcl</a>.
12:10:09 This server is running Tcl 8.4.8.
12:10:09 There is more <a href=version_history.html>TclHttpd version information</a>
12:10:09 about using other versions of Tcl and the Standard Tcl Library (tcllib).
12:10:09 <p>
12:10:09 
12:10:09 While this server works fine as a stand alone web server, the
12:10:09 intent is to embed the server in other applications in order to
12:10:09 "web enable" them.
12:10:09 A Tcl-based web server is ideal for embedding
12:10:09 because Tcl was designed from the start
12:10:09 to support embedding into other applications.
12:10:09 The interpreted nature of Tcl allows dynamic reconfiguration of the
12:10:09 server. Once the core interface between the web server and the hosting
12:10:09 application is defined, it is possible to manage the web server, upload
12:10:09 Safe-Tcl control scripts, download logging information, and otherwise
12:10:09 debug the Tcl part of the application without restarting the hosting
12:10:09 application.
12:10:09 <p>
12:10:09 
12:10:09   The server is distributed under a <a href=license.terms>copyright</a> that allows free use.
12:10:09

12:10:09 information about TclHttpd.

12:35:14 <p>
12:35:14 The TclHttpd is a pure Tcl implementation of a Web server.
12:35:14 It runs as a Tcl script on top of
12:35:14 <a href=http://www.tcl.tk/software/tcltk/>Tcl</a>.
12:35:14 This server is running Tcl 8.4.8.
12:35:14 There is more <a href=version_history.html>TclHttpd version information</a>
12:35:14 about using other versions of Tcl and the Standard Tcl Library (tcllib).
12:35:14 <p>
12:35:14 
12:35:14 While this server works fine as a stand alone web server, the
12:35:14 intent is to embed the server in other applications in order to
12:35:14 "web enable" them.
12:35:14 A Tcl-based web server is ideal for embedding
12:35:14 because Tcl was designed from the start
12:35:14 to support embedding into other applications.
12:35:14 The interpreted nature of Tcl allows dynamic reconfiguration of the
12:35:14 server. Once the core interface between the web server and the hosting
12:35:14 application is defined, it is possible to manage the web server, upload
12:35:14 Safe-Tcl control scripts, download logging information, and otherwise
12:35:14 debug the Tcl part of the application without restarting the hosting
12:35:14 application.
12:35:14 <p>
12:35:14 
12:35:14   The server is distributed under a <a href=license.terms>copyright</a> that allows free use.
12:35:14 
12:35:14 
12:35:14 <p>
12:35:14 The <a href="mailto:tclhttpd-users@lists.sourceforge.net">tclhttpd-users@lists.sourceforge.net</a> mailing list is the best source of current
12:35:14 information about TclHttpd.
